# PCI DSS Remediation Roadmap

**Planning baseline:** 2026-07-21 | **System:** LexHOA | **Standard:** PCI DSS v4.0.1

> This is a remediation plan, not a compliance finding, legal opinion, QSA certification, or attestation. Completing tasks in this roadmap does **not** itself establish current compliance; controls must operate, be evidenced, and be assessed. LexHOA's custom Accept.js page renders and handles payment fields before direct tokenization, so the AWS backend avoids intentional raw account-data storage but the browser application remains security-relevant.

## Step 0 — written scope and questionnaire confirmation (blocker)

The Compliance Owner must obtain written confirmation from the acquirer/payment brands, with QSA assistance as needed, of the merchant level, validation deadline, CDE/connected-to/security-impacting systems, service providers, and applicable questionnaire. **SAQ A-EP appears likely, but this roadmap never claims final eligibility.** Record why SAQ D or another validation form is or is not required, including treatment of eCheck, administrator endpoints, Amplify build/deploy, Git hosting, DNS, AWS, and Authorize.Net.

**Acceptance/evidence:** signed scope memo; system/component and data inventories; confirmed questionnaire/version; acquirer/QSA correspondence; current [network diagram](./network-diagram.svg), [payment flow](./payment-data-flow-diagram.svg), and [infrastructure narrative](./PCI_INFRASTRUCTURE.md), each approved and dated. **Dependencies:** acquirer and TPSP responses. **Maps:** 1.2.3–1.2.4, 12.1.3, 12.5.2, 12.8, questionnaire applicability.

## Recurring control calendar

- **Daily:** review critical security/payment alerts and designated audit events; reconcile failed/partial payment processing; open and track incidents.
- **Weekly:** run and review 11.6.1 payment-page/header tamper detection; review failed webhook/reconciliation and secret-scan alerts.
- **Monthly:** patch/dependency and vulnerability triage; account/key/service inventory drift; TPSP advisory review; restore/reconciliation sample.
- **Quarterly:** access review; external ASV scan and rescan if applicable; wireless/unauthorized-AP detection where the confirmed questionnaire requires it (absence of merchant wireless does not automatically make detection N/A under SAQ D); vulnerability review; evidence quality check.
- **Semiannual:** NSC/WAF/API ruleset review; incident contact and response-plan review; payment-page script inventory/authorization review; segmentation test if relied upon.
- **Annual:** scope/data-flow/network inventory; risk assessment; policies/RACI; training; incident exercise; penetration test and segmentation test as applicable; TPSP AOCs; questionnaire/AOC workflow and N/A revalidation.
- **After significant change:** scope/threat/risk review, approvals, security tests/scans, penetration test where required, diagram/inventory/script/CSP/tamper baseline updates, deploy evidence and training/runbook updates before closure.

Owners record evidence using [PCI Operations Manual](./PCI_OPERATIONS_MANUAL.md) conventions and update [PCI Questionnaire Answer Key](./PCI_QUESTIONNAIRE_ANSWER_KEY.md); roadmap status alone is never evidence that a control operates.
